iOS Quick Integration

The operation of the SDK requires appid and configuration files. When the merchant accesses, the test appid and test configuration file provided by the runcobo can be used first. When the official release, the official appid and configuration file must be obtained from the official.

Installation method

cocoapods installation:

  • Install Cocoapods first;
  • Update the cocoapods version of QNSDK via pod repo update;
  • Add pod 'QNSDK' to the target corresponding to the Podfile and execute pod install;
  • Run the project in the project using the .xcworkspace generated by CocoaPods;
  • Introduce the header file `#import in your code file header.

Carthage installation:

  • Install Carthage;
  • Open Cartfile and add github "https://github.com/YolandaQingniu/sdk-ios-demo.git";
  • Open the command line, cd to your project directory, enter carthage update;
  • Drag QNSDK.framework from the Carthage/Build/ directory to Build Phases -> Linked Binary and Libraries in your project configuration.
  • Introduce the header file `#import in your code file header.

Manual installation:

  • Download the SDK installation package to the project
  • Introducing SDK path [TARGETS]-> [Build Setting]->[Search Paths]->[LibrarySearch Paths]Add SDK path
  • Configure the linker [TARGETS]-> [Build Setting]-> [Linking]-> [Other Linker Flags] add one of -ObjC, -all_load, -force_load [SDK path]

Project configuration

  • There is a pair in the Info.plist [Privacy - Bluetooth Peripheral Usage Description] [Privacy - Bluetooth Always Usage Description] button.

Precautions

  • SDK adaptation 8.0 and above
  • iOS10.0 and above must have Bluetooth instructions in Info.plist, otherwise the system's Bluetooth function cannot be used.
  • iOS13.0 and above must be configured with the authorization instructions for Bluetooth in Info.plist, otherwise it will be found to crash.
  • The linker must be configured for the SDK or the SDK will not function properly
