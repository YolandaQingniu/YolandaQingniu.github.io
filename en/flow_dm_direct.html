WiFi+Bluetooth Dual Module Scale Integrate Guide

Introduction

The WiFi+Bluetooth dual module scale mentioned here refers to the body fat scale with two data transmission functions of WiFi and Bluetooth.

This document is write for the developer of customer use yolanda scale which direct send request to user cloud(server).

Connection Mode Action

Bluetooth

The ssid and password used to set up WiFi

Data transmission for weighing

WiFi

Weighing data transmission

WiFi Usage Process

For the First Time to Use

When first use，you need do something.

1. Scanning Device

Call the API of the normal Bluetooth scale, scan to the Qingniu Bluetooth device (QNBleDevice), and determine that deviceType is equal to SCALE_WIFI_BLE.

2. The configuration of WiFi

When the scale is before-delivery, ssid and password of WiFi are not saved at the scale end. At this time, relevant APP of Bluetooth should be called to configure the network at the scale end.

Distribution Network APIPlease check the distribution network of WiFi Bluetooth dual-mode devices.

3. Write network API

The Scale has been written to your request your server direct. So you need write the http api to handle the data from scale.

Network API

I assume that your server's address is: http://www.example.com/measure .

Your server will received a GET http request as: http://www.example.com/measure?code=1CBCEED257AFD63A598884B3A572429E

The 1CBCEED257AFD63A598884B3A572429E is encrypted hex string. You can decrypt with our lib for server. Our lib will generate the response.

There three type for encrypted hex string.

1. Synchronize the scale's time

This request may be received once a day and will be accompanied by a mac address. After receiving the request, it will return the result of the parsing library directly afterAfter processing with our parsing library

2. Upload the real time measurement data

It is real-time data uploaded When the user weighs using wifi connection instead of Bluetooth connection

3. Upload history measurement data

the weighing data will be saved in local and it will be upload once Wifi is connected with scale next time When the user weighs neither with Bluetooth nor connect Wifi

JAVA version

We just support java server for now. If your server is written in difference language, contact us.

We'll provide an .so file and .java file for you.

There is two method in our lib.

analysisScaleHexData

Analysis the hex string which uploaded from scale, and return a json string as result. We'll explain the json as below.

Return "" (empty string) if parsing fails

Parameter

hexString: String, Hexadecimal string which was uploaded from scale

generateBodyData

Generate the measurement data with hexString and user profile.

Parameter

hexString: String, Hexadecimal string which was uploaded from scale height: int, height,unit is cm. gender: int, gender, o is female, 1 is male. age: int, age

Result data

field name type description
type int the hex string type.
1:Synchronize the scale's time
2:Upload real time measurement data
3:Upload history measurement data
weight double user body weight. It will not set if type is 1
time int measuring time. Unix timestamp. Unit is ms. It will be current time if type is 1 or 2
mac string the scale's mac. It is the only ID for scale
res string the hex string which need return to scale
data array the measurement data. It is only appeared in generateBodyData. see body index

Example

1:Synchronize the scale's time

hex string: D8588ED60AFE8A8C16988694

Call analysisScaleHexData(hexString) you'll get json string like:

{"type":1,"res":"A15CEA4A700000A1","mac":"60:04:34:56:32:12","weight":0.00,"time":1559805008000}

There is no need call generateBodyData when type is 1.

2:Upload real time measurement data

hex string: D8588ED20AFE8A8C8FDC82BAA3DEAE8CDE984E94

Call analysisScaleHexData(hexString) you'll get json string like:

{"type":3,"res":"A200A2","mac":"60:04:34:56:32:12","weight":60.50,"time":1559804069000}

Provide user profile like: height = 172 cm, gender = 1 (male), age = 30.

Call generateBodyData(hexString,height,gender,age) you'll get json string like:

{"type":3,"res":"A30000710100CDE2","mac":"60:04:34:56:32:12","weight":60.50,"time":1559804069000,"measurement":[{"type":1,"value":60.50},{"type":2,"value":20.50},{"type":3,"value":11.30},{"type":4,"value":10.10},{"type":5,"value":3},{"type":6,"value":64.00},{"type":7,"value":57.30},{"type":8,"value":2.68},{"type":9,"value":1529.00},{"type":10,"value":4},{"type":11,"value":20.25},{"type":12,"value":53.64},{"type":13,"value":50.98},{"type":14,"value":28},{"type":16,"value":100},{"type":17,"value":4.20}]}

The type below data means some body data. Please see [body index](./attouched_list/body_indexes.md.

3:Upload history measurement data

hex string: D8588ED00AFE8A8C374C0A2CE29E90A3B09DC294A652A48C

Call analysisScaleHexData(hexString) you'll get json string like:

{"type":3,"res":"A200A2","mac":"60:04:34:56:32:12","weight":60.50,"time":1559804069000}

Provide user profile like: height = 172 cm, gender = 1 (male), age = 30. Call generateBodyData(hexString,height,gender,age) you'll get json string like:

{"type":3,"res":"A200A2","mac":"60:04:34:56:32:12","weight":60.50,"time":1559804069000,"measurement":[{"type":1,"value":60.50},{"type":2,"value":20.50},{"type":3,"value":11.30},{"type":4,"value":10.10},{"type":5,"value":3},{"type":6,"value":64.00},{"type":7,"value":57.30},{"type":8,"value":2.68},{"type":9,"value":1529.00},{"type":10,"value":4},{"type":11,"value":20.25},{"type":12,"value":53.64},{"type":13,"value":50.98},{"type":14,"value":28},{"type":16,"value":100},{"type":17,"value":4.20}]}

The type below data means some body data. Please see body index.
