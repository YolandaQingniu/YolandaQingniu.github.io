Frequently Asked Questions

common problem

How to determine the type of device

At present, the device types are only ordinary Bluetooth scale andBluetooth broadcast scale After invoking Bluetooth scan, common Bluetooth device will return inonDeviceDiscover, and bluetooth broadcast scale will be called back inonBroadcastDeviceDiscover.

In addition, it can also be simply judged by the Bluetooth name. The body fat scale with the Bluetooth name QN-Scale/QNScale1 is an ordinary Bluetooth scale, and the Bluetooth name with the QN-S3 is a Bluetooth broadcast scale.

What is appid and configuration file？What's the difference？

Appid is the unique identifier created by Qingniu for customers. It will be registered in our back office. A given appid usually does not change. The appid 123456789 in the demo is the appid that we test and let customers experience. The appid's characteristics are unstable and may be modified at any time. If a formal cooperation intention has been reached, it is recommended to apply to us for business or sales appid, the application process is fast, and it can be completed in a few minutes.

The configuration file is some encrypted data used in conjunction with the appid, which specifies the model. algorithm、Indicators and other information. Different customer requirements will be different, so we encapsulate the customer's customized requirements in this configuration file.

Can different clients use the same APPID

Android and iOS can use an appid, applet/applet plug-in cannot be the same as the appid of the SDK for the time being

Is the SDK offline？What data will it send to Light Cow Cloud？

In the method initSdk, the SDK will send a request to verify with our cloud whether the configuration file needs to be updated. If an update is needed, the latest configuration file will be downloaded (the configuration file is small, generally 512 bytes), and if no update is needed, no processing is done.

The SDK supports offline mode. If the customer removes the requirement, we will set the configuration of the item in the configuration file to offline. After being set offline, the SDK no longer sends any network requests (this can be verified with a packet capture tool). The configuration file is no longer updated. If you need to add a model or indicator, you need to apply for a configuration file with our company again, and then the developer Replace manually.

Can you tell if the scale is off?？

For ordinary Bluetooth scale anddual-mode scale, when no device is connected, the device object obtained by scanning QNBleDevice has an attribute isScreenOn, which indicates Whether the screen is bright. After connecting the device, you cannot directly determine whether the device is bright, but for some scales, the connection will be disconnected when the screen is held.

The isScreenOn attribute just indicates that the device is bright at the moment the SDK scans. The screen status of subsequent devices changes, and QNBleDevice does not update dynamically. Need to judge by scanning the new QNBleDevice callback.

For `broadcast scales', the device that can be scanned is the state that it is turned on, but it cannot be scanned, usually it is not turned on.

Missing data returned after measurement

If it is only the weight and BMI data, you can refer to No data such as body fat rate after measurement is completed.

If the body fat rate can be measured, but the indicator you want is missing, the situation is usually not included in the configuration file, you need to communicate with our business/sales to increase the indicator.

After our company increases the indicator, it will resend a configuration file, which needs to be replaced.

Send the model id of the scanned device to our company for confirmation

No data such as body fat rate after measurement

  1. Check if the information is correct, mainly birthday (used to calculate age). Whether the two parameters of height exceed the normal range
  2. Take off your shoes for measurement. When measuring with shoes, the bio-impedance cannot be measured, so indicators such as body fat rate cannot be calculated

What to do if the device cannot be scanned

  1. Check if Bluetooth is turned on. Is there a step on the scale, and whether the mobile phone is too far away from the scale (more than 10 meters)
  2. Check if the scanned device is connected by others
  3. If it is Android, you can check whether you have the positioning permission, and whether you have turned on the location service switch.
  4. Try restarting Bluetooth, no more, try restarting Bluetooth.

What is the storage data on the scale

When an ordinary Bluetooth scale is not connected to a mobile phone during weighing, the scale will store the data in the scale. When the next time it is connected to Bluetooth, it will transfer these data to the APP. Since the SDK cannot identify who the user is, it will send This data is assigned by the APP, and the assigned user can give the SDK to calculate the complete data.

Can you provide Bluetooth protocol

In principle we will not provide Bluetooth protocol, if you really need, you can communicate with our business

Why does the data measured by the SDK differ from the light cow or light cow health data?

We have several sets of algorithms, different algorithms that different devices may use.

In addition, our own APP has a health question mechanism that allows users to answer 2 questions, and uses different algorithms based on the user's answer.

The algorithm mechanism in the SDK is usually a unified algorithm that is immutable. That is to say, the same scale uses our APP to measure and uses the SDK to measure, the calculation data and weight are the same, there may be many differences. Therefore, our company does not recommend using the measurement data of our APP and SDK for comparison. Customers only need to care about whether the algorithm given is appropriate and stable.

How to troubleshoot the Bluetooth version of the online version

First check the user's mobile phone model, if possible, try to use the same model as the customer to test.

Register log output interface QNLogListener, record related logs (can save log files), and send them to our developers for analysis.

Can the SDK only do protocol analysis, Bluetooth scanning and connection are managed by the customer

This is achievable, and the implementation methods of ordinary Bluetooth scales and Bluetooth broadcast scales will be different.

It is technically difficult to implement Bluetooth scanning and connection management by yourself. This operation is not recommended if you have no relevant Bluetooth development experience.

Ordinary Bluetooth Scale

  1. Write your own Bluetooth protocol proxy class to implement each Bluetooth operation method
  2. Write Data Monitor Callback Class QNScaleDataListener and use [QNBleApi.setBleDeviceDiscoveryListener to register in SDK.
  3. Perform the Bluetooth scan by yourself, and pass the relevant information scanned to the SDK, using the method QNBleApi.buildDevice to create a Light Bull Bluetooth device object
  4. Create a user object, using the method QNBleApi.buildUser
  5. Create Bluetooth Protocol Processing Class Make a Bluetooth connection
  6. The connection is successful. After finding that the service is successful, call the prepare method of the Bluetooth protocol processing class
  7. Process the data of the QNScaleDataListener callback
  8. Handle after receiving stable data.

Bluetooth broadcast scale

When will the Bluetooth of the body fat scale be turned on, and how long will it turn off automatically after turning it on (or disconnect from the mobile phone)

Under normal circumstances, the body fat scale Bluetooth is turned on when the scale is stepped on (that is, the screen is lit), and the radio is transmitted. After the measurement is over, the scale will hold the screen. After the screen is closed, it usually connects for 1-2 minutes (depending on the production batch), and then actively disconnects the measurement. This mechanism is for the user to load the scale again in a short time without having to connect again. If the APP needs to be disconnected immediately after the measurement, it can actively disconnect after a delay of more than 1s after receiving stable data.

API of your official website《Definition of scale status》The status is 0-9. Why is it promising in the demo you provided?“-1”Status (and it does return to this status)

-1 is always present, indicating that the default state is lost, that is, the connection is disconnected. It is usually received after the scale actively disconnects. We will unify this state to the disconnected state as soon as possible.

After searching for Bluetooth and connecting it, keep your feet suspended on the body fat scale for a few seconds, then take your feet away, and return the status code of the measurement completion (there is no data returned in the method of returning data)

Why we can search for Bluetooth without any operation (no one is near the body fat scale within 10-30 minutes)

When weighing, if you connect to the app, the scale will cache this data, and it will broadcast for 5-30 minutes after the screen is closed (depending on the production batch, it varies). After the APP is connected at this time, these cached data can be received and displayed to the customer. This function is used to solve the problem. The user does not need a mobile phone when weighing. After the weighing is completed, he sits on the sofa and turns on the mobile phone to receive the data just now.
