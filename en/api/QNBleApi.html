method

initSdk

Initialize the SDK based on appid and verify the appid and configuration file. This method also loads the relevant configuration in the configuration file.

This method can be called only once, not multiple times. It is recommended that Android be called in Application.onCreate, and iOS should be called in the method of didFinishLaunchingWithOptions under AppDelegate. This method only performs lightweight operations, and time-consuming operations are performed asynchronously without affecting the startup speed of the APP.

Normally, this method sends a request to our server to update the configuration in the configuration file, such as updating the algorithm, updating the metrics, adding new models, and so on. This can achieve the effect of online upgrade configuration. If you do not want this feature explicitly, you can turn off this feature with our business communication (ie, the SDK is changed to offline mode). In addition, if there is no network, the request will not affect, the SDK will continue to use the last configuration.

Note: The 123456789 appid in the demo is the test id, and the test id has instability and random variability. After the agreement between the access provider and the runcobo, an independent and stable appid can be obtained. Remember that the test Id is not available for the online application market.

Parameters

Name Type Description
appId String app_id provided by the company to the client, verified by the local initialization packet, Android iOS shares the same appid
firstDataFile String The file path of the configuration file can be passed in the form of uri or the full path of the file. The file contains
callback QNResultCallback Android is the interface, IOS uses block or closure) callback method, returns the initialization result, error code reference attached table.

setBleDeviceDiscoveryListener

Set the scan callback object. When there is a scan result, it will be returned in the set listener method. The SDK only supports one callback object. Repeating the settings will overwrite the previous result. If you don't need to use it, you need to reset it to null, otherwise a memory leak may occur.

Parameters

Name Type Description
listener QNBleDeviceDiscoveryListener The listener contains a method that will be called back in this object when scanning to the device.

startBleDeviceDiscovery

Start scanning the Bluetooth device and return only to the device. Stops automatically when an error is encountered or Bluetooth is turned off.

The scan result is called back in QNBleDeviceDiscoveryListener

For some configuration behaviors of scanning, please refer to getConfig and QNConfig

Parameters

Name Type Description
callback QNResultCallback callback object, return to this call to start the scan is successful

stopBleDeviceDiscovery

Stop scanning the Bluetooth device. This method calls the system's stop scanning method. If the scanning is not started, you need to call the next stop scanning to ensure that there will be no more scanning object callback after calling this method.

Parameters

Name Type Description
callback QNResultCallback callback object, return to this call to stop scanning is successful

callback QNResultCallback Returns whether the connection operation is successful (not the connection is successful)

convertWeightWithTargetUnit

Converted to the value of the specified unit based on the weight of the supplied kg value

does not support st, if you pass st will directly return the value of lb, that is, pass lb or st return the same value, APP needs to directly convert the unit of stone, 1st = 14lb, such as 145.2lb will be displayed as 10 st 5.2 lb.

Parameters

Name Type Description
kgWeight double kg in kilograms
unit int 0 is kg, the default value
1 is lb, lb, all scales can support this unit
2 for jin, if the scale end does not support, it will display kg

return value

Type: double

Returns the value of the specified unit

buildUser

Create a user's factory method

Parameters

Name Type Description
userId String User unique identifier, cannot be empty, unique for each user, related to stored data
height int Height, unit cm, lowest 40, highest 240, beyond this range, callback error message.
gender String Sex, Male: male, Female: female, Callback error message when setting other values.
birthday Date Birthday, used to calculate age, date accuracy is accurate to the day, the calculated age range is 3~80, beyond this range, equal to the upper or lower limit. The date passed is greater than the current time, and the callback is incorrect.
athleteType (optional) int is the athlete mode (0 is the normal algorithm, 1 is the athlete algorithm), this field only takes effect when the age is greater than or equal to 18 years old
callback QNResultCallback Returns whether the operation of creating the user object is successful. If it is not successful, it returns the error code and the cause of the error.

return value

Type QNUser

Directly return the runcobo user object created according to the parameter, or return NULL if the creation is unsuccessful

Manage Bluetooth connection yourself. You need to have some understanding of the Bluetooth connection related API, use it with caution.

Parameters

connectDeviceSetWiFi

Distribution network for WiFi Bluetooth dual-mode devices

Before the distribution network, you need to determine whether the device has registered the device with the light cloud cloud. Otherwise, the data transmitted through the WiFi cannot be pushed to the cloud of the access side.

When using this method to distribute the network, the device will be automatically connected. When the distribution network ends, the accessor decides whether to disconnect.

Parameters

Name Type Description
device QNBleDevice Double-mode scale equipment with distribution network
user QNUser User model used, including userId, and user profile
wifiConfig QNWiFiConfig With the wifi information of the network
callback QNResultCallback Returns whether the distribution operation is successful (not the distribution result, please check the ERROR_DEVICE_TYPE in Scale Status Definition),ERROR_WIFI_PARAMS, ERROR_REGISTER_DEVICE)
