QNBleApi

Entrance class

Method

initSdk

Initialize the SDK according to appid and verify the appid and configuration file. This method also loads the relevant configuration in the configuration file.

This method only needs to be called once, not multiple times. Android is recommended to be called in Application.onCreate, and iOS is to be called in didFinishLaunchingWithOptions under AppDelegate. This method only performs lightweight operations, time-consuming operations are performed asynchronously, and it does not affect the startup speed of the APP.

Under normal circumstances, this method will send a request to our server to update the configuration in the configuration file, such as updating algorithms, updating indicators, adding new models, etc. This can achieve the effect of online upgrade configuration. If you do not want to have this feature, you can turn off the function (that is, the SDK is changed to offline mode) by communicating with our company. In addition, if there is no network, there will be no impact after the request fails, and the SDK will continue to use the previous configuration.

Note: The 123456789 appid in the demo is the test id. The test id has instability and random variability. After the access provider and Lightniow reach an agreement, they can obtain independent and stable appids. Remember that the test Id cannot be used in the online application market

parameters

name Types of Description
appId String The app_id provided by the company to the customer is verified through the local initialization packet. Android iOS shares the same appid
firstDataFile String The file path of the configuration file can be passed in the form of uri or the full path of the file. The file contains
callback QNResultCallback Android is an interface. IOS uses a block or closure) callback method, which returns the result of initialization. See the attached table for error codes.

setBleDeviceDiscoveryListener

Set the scan callback object. When there is a scan result, it will be returned in the set monitoring method. The SDK supports only one callback object. Repeated settings will overwrite the previous results. When it is not needed, it needs to be reset to null, otherwise a memory leak may occur.

parameters

name Types of Description
listener QNBleDeviceDiscoveryListener The listener contains a method that will callback in this object when it scans the device.

startBleDeviceDiscovery

Start scanning for Bluetooth devices, only returning devices that are light cows. It will stop automatically when it encounters an error or Bluetooth is turned off.

Scan results are called back in QNBleDeviceDiscoveryListener

For some configuration behaviors of scanning, please refer to getConfig and QNConfig

parameters

name Types of Description
callback QNResultCallback Callback object, returns whether the call to start the scan is successful

stopBleDeviceDiscovery

Stop scanning the Bluetooth device. This method calls the system's stop scanning method. If the scan is not started, it also needs to be called to stop scanning to ensure that there will be no scan object callback after calling this method.

parameters

name Types of Description
callback QNResultCallback Callback object, returns whether this call to stop scanning is successful

connectDevice

Connect the Light Cow Bluetooth device. The connection process will be called back in QNBleConnectionChangeListener, and the measurement data will be called back in QNScaleDataListener

parameters

name Types of Description
device QNBleDevice Requires a connected Bluetooth device.
user QNUser When connecting devices, the user model used includes userId and user profile. Created by buildUser
callback QNResultCallback Returns whether the connection operation is called successfully (not the connection is successful)

disconnectDevice

Disconnect the connected Bluetooth device. The disconnection process will be called back in QNBleConnectionChangeListener

parameters

name Types of Description
device/mac QNBleDevice/String Need to disconnect the Bluetooth device or mac address, just wear one of these two parameters
callback QNResultCallback Returns whether the connection operation is called successfully (not the connection is successful)

registerWiFiBleDevice

Register WiFi dual-mode device with Qingniu Cloud

The operation of registering the device with the Light Cow Cloud is to upload the data from the scale to the Light Cow Cloud directly via WiFi, and the Light Cow Cloud can accurately identify the data and transfer the relevant measurement information to the access party's cloud.

This method only supports the use of WiFi Bluetooth dual-mode devices

parameters

name Types of Description
device QNBleDevice Devices to be registered
callback QNResultCallback Back to registration results

return value

Types of: QNConfig

convertWeightWithTargetUnit

Convert the weight to the specified unit based on the weight provided in kg

St is not supported. If you pass st, it will directly return the value of lb, that is, if you pass lb or st, the same value will be returned. The app needs to directly convert the unit of yingshi, 1st = 14lb. For example, 145.2lb will be displayed as 10 st 5.2 lb.

parameters

name Types of Description
kgWeight double Kilogram weight
unit int 0 is kg, default
1 is lb, all scales can support this unit
2 is kg, if the scale end does not support, kg will be displayed

return value

Type: double

Returns the value of the specified unit

buildUser

Factory method for creating users

parameters

name Types of Description
userId String User unique identifier, cannot be empty, unique for each user, related to stored data
height int Height, unit cm, minimum 40, maximum 240, beyond this range, error message will be called back.
gender String Sex: Male:male, female:female, callback error message when setting other values.
birthday Date The birthday is used to calculate the age. The accuracy of the date is accurate to the day. The calculated age range is 3 ~ 80. If it exceeds this range, it is equal to the upper or lower limit. The date passed is greater than the current time, and the callback error occurs.
athleteType (optional) int Whether it is athlete mode (0 is ordinary algorithm, 1 is athlete algorithm), this field will only take effect when the age is greater than or equal to 18
callback QNResultCallback Returns whether the operation of creating a user object is successful. If it is unsuccessful, an error code and an error reason are returned.

return value

Type QNUser

Return the Light Cow user object created according to the parameters directly, or NULL if the creation is not successful

buildProtocolHandler

Create a Bluetooth protocol processor to help customers manage Bluetooth connections themselves

Managing the Bluetooth connection yourself requires a certain understanding of the Bluetooth connection-related APIs, use it with caution

parameters

name Types of Description
device QNBleDevice The scale device to be connected can be created with buildDevice
user QNUser User model used, can be created with buildUser
delegate QNBleProtocolDelegate Auxiliary class for protocol processing class to assist in operating Bluetooth
callback QNResultCallback Whether the return operation was successful

return value

Type QNBleProtocolHandler

If the creation fails, null/nil is returned. And return error code in callback.

connectDeviceSetWiFi

Networking of WiFi Bluetooth dual-mode devices

Before configuring the network, you must first determine whether the device has registered with Lightniu Cloud, otherwise the data transmitted through WiFi cannot be pushed to the cloud of the access party.

When using this method to configure the network, the device will be automatically connected. When the network is finished, the access party determines whether to disconnect.

parameters

name Types of Description
device QNBleDevice Need dual network scale equipment
user QNUser User model used, including userId, and user profile
wifiConfig QNWiFiConfig Wifi information
callback QNResultCallback Returns the success of the distribution network operation (not the distribution network result, please refer to the ERROR_DEVICE_TYPE,ERROR_WIFI_PARAMS, ERROR_REGISTER_DEVICE in Scale Status Definition for the distribution network result
