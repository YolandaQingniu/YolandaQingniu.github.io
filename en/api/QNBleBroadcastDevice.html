QNBleBroadcastDevice

QNBleBroadcastDevice

Light cattle broadcast scale Bluetooth device object

A broadcast scale is a device that acquires relevant data through a broadcast sent by a receiving device without connection. The scale is different from the ordinary Bluetooth scale. The difference is that the ordinary Bluetooth scale can initiate a connection to a surrounding device when scanning the surrounding device information, so that it can choose whether to stop scanning the surrounding device information without affecting the connected device. Get relevant measurement information.

However, since the broadcast scale analyzes related measurement information by acquiring broadcast data of surrounding devices, it is not necessary to connect a certain device. Therefore, if the scanning of the surrounding equipment is stopped during the measurement using the broadcast scale, it means that the measurement data of this time cannot be obtained.

Attributes

Name Type Description
mac String The MAC address, sdk will be parsed according to the scanned broadcast information. The same device, Android and IOS return the same result. Unique per device
name String Model name, the SDK will identify the scanned device model, if it is not recognized, return a default model name, Scale
modeId String Model Identification
bluetoothName String Bluetooth name, this is the hardware device broadcast name, different from name. APP is based on name.
RSSI int Signal strength, is a negative number, generally between -30~-100. The larger the value, the greater the signal strength. The same device may call back multiple times. The value of the signal strength may be used each time the callback is made. Different.
supportUnitChange boolean Whether to support the protocol mode to modify the scale unit
unit int The unit currently displayed by the scale body
0 is kg, the default value is />1 is lb
2 for jin
weight double The weight value displayed by the current scale body
isComplete boolean whether the measurement is complete
measureCode int The data is encoded after the measurement is completed. After the measurement is completed, multiple identical broadcast data may be received. Therefore, this field is used to distinguish whether multiple measurement completion data received in the adjacent time is the same measurement data.

syncUnit

When the supportUnitChange field of the device is true, it indicates that the device is allowed to be modified by means of a protocol. In this case, you can use this method to modify the unit.

When the method is successfully called, the SDK will send a Bluetooth broadcast signal. When the target device unit has been modified or 10s, the SDK will stop broadcasting.

After the method is called, the operation of the previous synchronization unit will be canceled, for example, the unit of the A device is being synchronized. When the unit of the B device is synchronized, the SDK will cancel the synchronization of the A device unit first, and then start the unit synchronization of the B device.

This unit is the unit set in QNConfig
