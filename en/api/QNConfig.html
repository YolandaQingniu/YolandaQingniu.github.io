QNConfig

QNConfig

SDK configuration object, which controls the scanning behavior, and sets some of the SDK's behavior through it

Attributes

name Types of Description
onlyScreenOn Boolean Whether to return only the devices that are turned on (bright screen). The default is false, that is, the scales that are turned on and not turned on.
allowDuplicates Boolean Whether the same device returns multiple times. Generally speaking, the device will always send Bluetooth broadcasts. The system will callback the same device multiple times during scanning. The SDK will process the same device, so that during a scan, only one device returns. Once, the merchant can set this parameter, and the SDK returns directly every time. Default false (this field is invalid for onBroadcastDeviceDiscover monitoring)
duration int Scan duration in ms. The default is 0, that is, scanning is performed unless the APP calls stopBleDeviceDiscovery. When it is not 0, the minimum value is 3000ms, and the scan will automatically stop after a delay of ms. (If there is access to the broadcast scale, please pay attention to the setting of this field, see Broadcast Scale Working Principle)
connectOutTime long (Android exclusive) Limited time for connecting the device, unit is ms, the default is 10000, that is, within 10 seconds of calling the connection, if the entire connection process cannot be completed, the connection will be stopped and callback error;When the value is less than or equal to 0, it means no callback;When the value is less than 3000 and greater than 0, it indicates that the timeout is 3000ms. If the connection has been stopped before the connection timed out, there will be no error callback even if the device is not successfully connected during the period.
unit int 0 is kg, default
1 is lb, all scales can support this unit
2 is kg, if the scale end does not support, kg will be displayed
3 is st, Yingshi, if the scale end does not support, the value of lb, will be displayed
showPowerAlertKey bool The default is false. When true, when the SDK is initialized and Bluetooth is turned off, the system will pop up the Bluetooth status. For details, please refer to Apple Developer Documenttation => CoreBluetooth => CBCentralManager => Central Manager Initialization Options
setNotCheckGPS bool (Android exclusive) The default is false. When true, the SDK will not perform GPS permission judgment when scanning. Continue to scan
enhanceBleBoradcast bool Whether the enhanced broadcast scale signal needs to be activated during scanning (not valid for ordinary scales, this option is only for broadcast scales)

unit

The unit displayed on the terminal. If it is not set, the SDK defaults to kg. After setting, it will be saved locally. If there is already a connected device, the unit display on the scale will be updated in real time.

The SDK will always return the value of kg. The APP needs to perform numerical conversion by itself.

Method

save

Save modified settings

parameters

name Types of Description
callback QNResultCallback Returns whether the setting operation was successful
